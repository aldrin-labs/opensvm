# OpenSVM API Specification v1.0
# Comprehensive API Documentation for LLMs and Developers
# Generated: 2025-09-26

================================================================================
## EXECUTIVE SUMMARY
================================================================================

OpenSVM provides a comprehensive REST API for interacting with the Solana blockchain.
The API offers 85+ endpoints covering transactions, accounts, analytics, tokens, NFTs,
and real-time data feeds. Built on Next.js 15 with App Router.

Base URL: https://osvm.ai/api (production)
          http://localhost:3000/api (development)

API Version: v1
Rate Limits: 60 requests/minute per IP (general), endpoint-specific limits apply
Response Format: JSON (application/json)
Timeout: 10 seconds default, 2-5 seconds for analytics

================================================================================
## AUTHENTICATION MECHANISMS
================================================================================

### 1. JWT Authentication (Bearer Token)
Used for: Monetization endpoints, API management
Header: Authorization: Bearer <token>
Endpoints: /api/opensvm/*, /api/v1/models

### 2. Wallet Signature Authentication
Used for: User authentication, social features
Process: Sign message with Solana wallet, verify on server
Endpoints: /api/auth/verify, /api/auth/session

### 3. API Key Authentication
Used for: External API access
Header: X-API-Key: <key>
Management: /api/opensvm/anthropic-keys/*

### 4. Session-Based Authentication
Cookie: svmai_session
Duration: 30 days
Auto-generated for anonymous users

================================================================================
## ERROR CODES
================================================================================

400 - Bad Request: Invalid parameters
401 - Unauthorized: Missing or invalid authentication
403 - Forbidden: Insufficient permissions
404 - Not Found: Resource not found
429 - Rate Limited: Too many requests
500 - Internal Error: Server error
502 - Bad Gateway: Upstream service error
503 - Service Unavailable: Temporary outage
504 - Gateway Timeout: Request timeout

### GET /api/trades
Fetch recent trades/swaps for any Solana token with wallet addresses and complete trade information

Request Parameters:
  - mint: string (required, query) - Token mint address
  - limit?: number (query) - Number of trades to return (default: 50, max: 100)
  - type?: string (query) - Transaction type filter (default: 'swap')
    Options: 'swap' (token swaps), 'add' (liquidity additions), 'remove' (liquidity removals)
  - offset?: number (query) - Pagination offset for fetching more results (default: 0)

Request Schema:
GET /api/trades?mint={TOKEN_MINT}&limit={LIMIT}&type={TYPE}&offset={OFFSET}

Response Schema (200 OK):
{
  "success": true,
  "mint": string,
  "trades": [
    {
      "signature": string,        // Transaction signature
      "timestamp": number,        // Unix timestamp
      "blockTime": number,        // Block timestamp
      "type": string,             // Transaction type: "swap", "add", "remove"
      "side": string,             // Trade side: "buy" or "sell"
      "price": number,            // Price per token
      "priceUSD": number,         // Price in USD
      "amount": number,           // Token amount traded
      "amountUSD": number,        // USD value of trade
      "volume": number,           // Volume (same as amountUSD)
      "token": {
        "symbol": string,
        "name": string,
        "address": string,       // Token mint address
        "decimals": number
      },
      "pairToken": {
        "symbol": string,        // Usually "USDC", "SOL", etc.
        "name": string,
        "address": string,       // Pair token mint address
        "decimals": number
      },
      "dex": string,             // DEX name: "Raydium", "Orca", "Phoenix", "Meteora", etc.
      "poolAddress": string,     // AMM pool address
      "owner": string,           // Wallet address that made the trade
      "slot": number,            // Solana slot number
      "raw": {                   // Raw Birdeye response data
        "base": object,
        "quote": object,
        "pricePair": number
      }
    }
  ],
  "count": number,               // Number of trades in response
  "total": number,               // Total trades available (if known)
  "metadata": {
    "mint": string,
    "limit": number,
    "offset": number,
    "txType": string,            // Filter type used
    "hasMore": boolean          // Whether more trades are available
  }
}

Error Response (400/500):
{
  "error": string,
  "usage": string,
  "details"?: any
}

Response Fields Explained:
- signature: Solana transaction signature for verification
- owner: CRITICAL - Wallet address of trader (for tracking whale activity)
- side: Automatically detected based on token flow ("buy" = acquiring target token)
- dex: DEX where trade occurred (useful for DEX activity analysis)
- priceUSD: Calculated USD value at time of trade
- amountUSD: Total USD value of the trade
- raw: Complete Birdeye response for advanced use cases

Features:
- Real-time trading activity tracking
- Wallet address tracking in "owner" field for whale watching
- DEX identification (Raydium, Orca, Phoenix, Meteora, Jupiter, etc.)
- Pagination support for large datasets
- Trade side detection (buy/sell) based on token flow
- USD value calculations for all trades
- Complete raw data for advanced analysis
- Filter by transaction type (swaps, liquidity adds/removes)

Use Cases:
1. Real-time trade feeds for tokens
2. Whale wallet tracking via owner field
3. Trading history and pattern analysis
4. DEX activity distribution analysis
5. Large trade detection and alerts
6. Volume analysis by wallet or DEX
7. Price impact analysis

Example Requests:
- Last 50 trades: /api/trades?mint=DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263
- Last 100 trades: /api/trades?mint=DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263&limit=100
- Only swaps: /api/trades?mint=DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263&type=swap
- Paginated (page 2): /api/trades?mint=DezXAZ8z7PnrnRJjz3wXBoRgixCa6xjnB7YaB1pPB263&limit=50&offset=50

Cache: 30 seconds
Rate Limit: 30 req/min

================================================================================
## RATE LIMITING
================================================================================

Default: 60 requests/minute per IP
Analytics: 30 requests/minute
AI Analysis: 10 requests/minute
Batch Operations: 10 requests/minute
Real-time Feeds: 1 connection per IP

Headers returned:
- X-RateLimit-Limit: Maximum requests
- X-RateLimit-Remaining: Requests remaining
- X-RateLimit-Reset: Reset timestamp

================================================================================
## CACHING
================================================================================

Default cache times:
- Transaction data: 5 minutes
- Account stats: 5 minutes
- Analytics: 2 minutes
- Token metadata: 30 minutes
- Block data: Permanent
- Search results: 1 minute

Cache headers:
- Cache-Control: public, max-age=300
- ETag: For conditional requests
- Last-Modified: Resource modification time

================================================================================
## WEBSOCKET ALTERNATIVES
================================================================================

OpenSVM uses Server-Sent Events (SSE) instead of WebSockets for real-time data.
Benefits:
- Works through proxies/firewalls
- Auto-reconnection
- Simpler implementation
- HTTP/2 compatible

Connection: GET request with Accept: text/event-stream
Keepalive: Every 30 seconds
Reconnect: Automatic with exponential backoff
